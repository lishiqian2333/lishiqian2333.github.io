solana学习笔记

CIL

  • 创建新地址

    solana-keygen new —outfile

  • 输出从私钥文件转换出的公钥地址 默认会从solana配置文件的设定找到私钥文件

    solana-keygen pubkey

  • 获得solana配置文件

    solana config get

  • 设置为开发者网络

    solana config set —url https://api.devnet.solana.com

  • 获取空投

    solana airdrop 1 —url https://api.devnet.solana.com

  • 查询余额

    solana balance —url https://api.devnet.solana.com

  • 转账

    solana transfer —from 0.5 —allow-unfunded-recipient —url https://api.devnet.solana.com —fee-payer

    solana transfer —from —fee-payer

  • 部署合约

    solana program deploy

  • 查询账户地址

    solana program show

solana相关概念

  • 使用Rust、C、C++，在solana上部署合约。任何人均可使用。
  • 通过SDK编写dapp与合约通信

交易

solana中的基本活动单位，需要通过交易来改变链上数据，但是可以不通过交易获取链上数据

交易结构：

  • Signatures（签名）：交易中所有签名者的签名，是一个数组（数字签名采用 ed25519 二进制格式，占用 64 个字节）
  • 消息：
    • Header（消息头）：描述有多少地址会进行签名，多少不会，以及多少只读（three unsigned 8-bit values）
    • Account addresses：交易中使用的账户，是一个数组。需要签名的地址显示在帐户地址数组的开头，首先请求读写访问权限的地址，然后是只读帐户。不需要签名的地址跟在需要签名的地址后面，同样是读写帐户在前，只读帐户在后。
    • Recent blockhash：最近的区块哈希（ 32 字节的hash）
    • Instructions：交易的具体指令，要调用哪个程序、要使用的帐户以及程序执行指令所需的任何其他数据。
      指令包含一个程序 ID 索引，后跟一个帐户地址索引的紧凑数组，后跟一个不透明 8 位数据的紧凑数组

    关于PDA

PDA（程序派生地址），特定程序控制的帐户，由program_id，seed参与计算生成，owner为program。由于PDA不在椭圆曲线上，因此无法有一对keypair能对其生成生成签名,只能通过程序来生成签名。

关于find_program_address

可以看到传入seeds（seeds的选择取决于程序如何计算派生），program_id可以进行计算一个有效的pda与其对应的bump种子

关于invoke_signed

程序如何为PDA生成签名？程序可以通过调用 invoke_signed 并传递用于生成地址的相同种子以及计算出的 bump 种子来“签名”，过程中会再次计算派生地址，如果匹配，那么就等于生成了签名。
